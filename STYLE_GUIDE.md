# Crowdcode Style Guide

This project should follow a clean, modern UI direction inspired by shadcn/ui.

## Core rules
- Use shadcn-style visual patterns: neutral backgrounds, soft borders, subtle shadows, rounded corners, and clear typography.
- Avoid loud or gimmicky UI choices.
- Do not use gradients for backgrounds, cards, buttons, or text treatments.
- Prefer solid colors, borders, and restrained emphasis.
- Keep the interface simple, accessible, and production-like.

## Visual direction
- Backgrounds: white, slate, zinc, gray, or near-black neutrals.
- Surfaces: cards with border + light shadow, not heavy glow effects.
- Buttons: solid primary color, clear hover state, no gradient fills.
- Typography: strong hierarchy, readable sizes, consistent spacing.
- Motion: subtle transitions only.

## Component guidance
- Use card-based layouts for grouped content.
- Use rounded corners similar to shadcn/ui.
- Keep spacing consistent with a simple scale.
- Ensure all interactive elements have accessible hover/focus states.
- Prefer utility-like styling patterns and reusable class names.

## Coding agent hints
- When adding or changing UI, aim for a shadcn/ui look and feel.
- If something feels flashy, simplify it.
- If a design choice requires a gradient, replace it with a solid color or border treatment.
- Preserve readability and polish over decoration.
- Default to minimal, modern, and functional.

## Current project intent
This project is a small demo site, but future changes should still follow the same design language so it can grow into a consistent shadcn-based interface.
